## 1. ภาพรวม

BoxBox ช่วยจัดการกล่องยาฉุกเฉิน (CPR, PPH, ACS, EMS ฯลฯ) ทุกกล่องในโรงพยาบาล ตั้งแต่การบรรจุยา การจ่ายกล่องออกตึก ไปจนถึงการแลกกล่องและรายงานสรุป

### วงจรชีวิตกล่อง

```
[สร้างกล่อง] → filling → (บรรจุยา) → ready → dispatched → filling → ...
                                                          ↘ retired
```

| สถานะ | ความหมาย |
|-------|----------|
| `filling` | กำลังเตรียมยา / รอบรรจุ |
| `ready` | บรรจุแล้ว พร้อมจ่ายออกตึก |
| `dispatched` | จ่ายออกตึกแล้ว |
| `retired` | เลิกใช้งาน |

## 2. Dashboard

หน้าหลักของแอป แสดงสถานะกล่องทุกใบแบบ real-time

### 2.1 KPI Cards (แถวบนสุด)

| การ์ด | ความหมาย |
|------|----------|
| กล่องทั้งหมด | จำนวนกล่องที่ยังใช้งานอยู่ทั้งหมด |
| พร้อมจ่าย | กล่องที่บรรจุเสร็จแล้ว รอจ่ายออกตึก |
| จ่ายออก | กล่องที่อยู่ที่ตึกผู้ป่วยแล้ว |
| กำลังเตรียม | กล่องที่อยู่ระหว่างบรรจุยา |
| หมดอายุ | กล่องที่ยาหมดอายุแล้ว (แจ้งเตือนสีแดง) |
| ใกล้หมดอายุ | กล่องที่ยาจะหมดภายในเกณฑ์ที่ตั้ง |
| ยืนยันแล้ว | กล่องที่ตึกยืนยันรับเรียบร้อยแล้ว |

### 2.2 ตัวกรอง (Filter Bar)

- **หมวดหมู่** — กรองตาม CPR / สูติกรรม / ฉุกเฉิน / EMS
- **ประเภทกล่อง** — กรองตามชนิดกล่อง (CPR Adult, PPH, ACS ฯลฯ)
- **ตึก / Ward** — กรองตามหอผู้ป่วย
- สามารถเลือกหลายรายการได้พร้อมกัน

### 2.3 Card View (มุมมองการ์ด)

แต่ละการ์ดแสดง:
- รหัสกล่อง, ประเภท, หมวดหมู่
- สถานะ (badge สี)
- วันหมดอายุกล่อง / วันที่บรรจุล่าสุด
- ตึกที่กล่องอยู่
- ปุ่ม `⋯` เมนูการดำเนินการ

**เมนูบนการ์ด:**

| เมนู | ใช้ทำ |
|-----|------|
| บรรจุยา | เปิด FillModal เพื่อบรรจุยาและจ่ายกล่อง |
| ประวัติกล่อง | ดูประวัติการบรรจุ/จ่ายของกล่องนี้ |
| ลบกล่อง | ซ่อนกล่อง (soft-delete, ไม่ลบจริง) |

### 2.4 Pivot View (มุมมองตาราง)

คลิกปุ่ม **ตาราง** เพื่อสลับเป็น pivot view แสดงสถานะกล่องตามตึกและประเภทในรูปตาราง  
คลิก cell ใดก็ได้เพื่อกลับไป Card View กรองเฉพาะ ward + type นั้น

### 2.5 Bulk Dispatch

เลือกหลายกล่องพร้อมกัน แล้วจ่ายออกตึกเดียวกันในครั้งเดียว:
1. คลิก checkbox บนการ์ด (หรือ "เลือกทั้งหมด")
2. เลือกตึกปลายทางจาก dropdown
3. กด **จ่ายกล่องที่เลือก**

## 3. บรรจุยา (FillModal)

เปิดจากเมนู `⋯ → บรรจุยา` บนการ์ดกล่อง — มี 3 ขั้นตอน

### ขั้นตอนที่ 1 — บรรจุยา

| ฟิลด์ | คำอธิบาย |
|------|---------|
| วันหมดอายุ (EXP) | พิมพ์หรือเลือกจาก calendar — แสดงรูปแบบ วว-ดด-ปปปป |
| Lot No. | เลขล็อตจากบรรจุภัณฑ์ยา |
| จำนวน (qty) | จำนวนที่บรรจุจริง |
| หมายเหตุ | บันทึกเพิ่มเติมรายรายการ |

**ไอคอนช่วยเหลือ:**
- 🕐 — ค่า prefilled จากการบรรจุครั้งก่อน (คลิกค้างเพื่อดูวันที่)
- ★ — ค้นพบจาก INVS historical match (ความน่าเชื่อถือสูงสุด)
- 🔍 — คลิกเพื่อค้นหา Lot/EXP จากระบบ INVS โดยตรง

**Prefill Source:** เลือกว่าจะดึงค่าเดิมจาก "กล่องนี้" หรือ "กล่องทุกใบประเภทเดียวกัน"

**ค้นหา INVS:**  
คลิกไอคอน 🔍 ที่รายการยา ระบบจะค้นหาจากคลัง INVS อัตโนมัติ  
ผลลัพธ์เรียงตามคะแนน Smart Match — เลือก Lot ที่ต้องการเพื่อเติมข้อมูล

### ขั้นตอนที่ 2 — เลือกกล่องแทน

เลือกกล่องประเภทเดียวกันที่มีสถานะ `ready` เพื่อจ่ายออกตึกแทนกล่องเดิม  
- แสดงวันหมดอายุของแต่ละตัวเลือก
- ตึกปลายทางจะถูกตั้งตามกล่องที่รับเข้า

### ขั้นตอนที่ 3 — พิมพ์และยืนยัน

- เลือก **ผู้เตรียมยา** (เจ้าหน้าที่เทคนิค) และ **เภสัชกรตรวจสอบ**
- ปุ่ม **พิมพ์ฉลากกล่อง** — พิมพ์สติกเกอร์ติดกล่อง
- ปุ่ม **พิมพ์ใบปะหน้ากล่อง** — พิมพ์ใบรายการยาในกล่อง
- ปุ่ม **ยืนยันและบันทึก** — บันทึกการบรรจุ กล่องเดิมเปลี่ยนเป็น `filling`, กล่องใหม่เปลี่ยนเป็น `dispatched`

## 4. แลกกล่อง (Exchange)

ใช้เมื่อต้องการรับกล่องที่ใช้แล้วคืนจากตึก และจ่ายกล่องใหม่แทน

### 4.1 ฟอร์มแลกกล่อง

1. **เลือกกล่องที่รับคืน** — dropdown แสดงเฉพาะกล่องที่อยู่ตึก (status = `dispatched`)
2. **บันทึกยาที่เหลือ** — ระบุจำนวนยาที่คืนกลับมาจริง (ถ้าต่างจากที่จ่ายออก)
3. **เลือกกล่องที่จ่ายแทน** — กล่องประเภทเดียวกัน status = `ready`
4. **ผู้ดำเนินการ** — ชื่อเจ้าหน้าที่
5. กด **แลกกล่อง** เพื่อยืนยัน

ผล: กล่องที่รับคืน → `filling`, กล่องที่จ่ายแทน → `dispatched` (ตึกเดิม)

### 4.2 ประวัติการแลกกล่อง

คลิกแท็บ **ประวัติ** เพื่อดูรายการแลกกล่องทั้งหมด พร้อมวันเวลาและผู้ดำเนินการ

## 5. รายงาน (Reports)

เมนูซ้ายมีรายงาน 9 หมวด:

### 5.1 ภาพรวม (Overview)

- KPI cards สรุปสถานะทั้งหมด
- ตารางสถานะตามประเภทกล่อง
- แจ้งเตือนกล่องหมดอายุ / รอยืนยัน (คลิกไปยังรายงานที่เกี่ยวข้องได้)

### 5.2 ประวัติการบรรจุ (Fill History)

ค้นหาและกรองประวัติการบรรจุยา:
- กรองตามช่วงวันที่, ตึก
- แสดง: รหัสกล่อง, ประเภท, วันบรรจุ, ผู้เตรียม, เภสัชกร, วันหมดอายุกล่อง
- คลิกแต่ละแถวเพื่อดูรายการยาทั้งหมดในการบรรจุนั้น

### 5.3 กล่องหมดอายุ

รายการกล่องที่ยาหมดอายุแล้ว พร้อมจำนวนวันที่เกินกำหนด

### 5.4 ใกล้หมดอายุ

สลับ mode ดูได้ 2 แบบ:
- **มุมมองกล่อง** — แสดงกล่องที่ใกล้หมดอายุ กรองตาม ward
- **มุมมองยา** — แสดงยาแต่ละตัวที่ใกล้หมดอายุ กรองตามช่วงวันที่และ ward

### 5.5 การใช้ยา (Drug Usage)

ตารางสรุปการใช้ยา:
- กรองตามช่วงวันที่ และ ward
- แสดง: ชื่อยา, จำนวนที่จ่ายออก, จำนวนที่คืน, ยอดใช้จริง

### 5.6 ค้นหา Lot (Lot Recall)

ค้นหากล่องตามเลข Lot number:
- พิมพ์ Lot No. ในช่องค้นหา
- แสดงทุกกล่องที่เคยใช้ Lot นั้น พร้อมวันที่บรรจุและตึกที่จ่ายออก

### 5.7 พร้อมจ่าย / ยืนยันแล้ว

ดูกล่องที่:
- **ยืนยันแล้ว** — ตึกสแกน QR ยืนยันรับกล่องแล้ว
- **รอยืนยัน** — จ่ายออกแต่ยังไม่ได้ยืนยัน

### 5.8 รายงานประจำเดือน (Monthly Report)

เลือกเดือน/ปี แสดง:
- จำนวนการบรรจุ, กล่องที่จ่าย, Lot ที่ใช้, ยาที่บรรจุ (หน่วย)
- ตารางรายการบรรจุทั้งหมดในเดือนนั้น

### 5.9 ประวัติแจ้งเตือน LINE

ดูประวัติการส่ง LINE Notify พร้อมสถานะ (สำเร็จ / ล้มเหลว / รอ GAS)

## 6. แจ้งเตือน LINE Notify

แท็บ **การแจ้งเตือน** แสดง:
- สถิติการส่ง: ทั้งหมด / สำเร็จ / ซ้ำ
- รายการแจ้งเตือนพร้อม filter (วันนี้ / Direct / GAS / สำเร็จ / ซ้ำ)
- ปุ่ม **Run Now** — ตรวจสอบและส่งแจ้งเตือนทันที

**ระดับแจ้งเตือน:**
- 🔴 วิกฤต — หมดอายุภายใน X วัน (ตาม alertRed)
- 🟡 ระวัง — หมดอายุภายใน Y วัน (ตาม alertYellow)
- ⚫ หมดอายุ — เกินวันกำหนดแล้ว

การส่ง LINE มี 2 ช่องทาง (ทำงานพร้อมกันถ้าตั้งค่าไว้):
- **Direct** — ส่งตรงจากแอปผ่าน LINE Notify API
- **GAS Trigger** — ส่งผ่าน Google Apps Script ตามเวลาที่ตั้ง

## 7. การตั้งค่า (Settings)

เข้าผ่านแท็บ **⚙️ ตั้งค่า** — sidebar ซ้ายแบ่งเป็น 3 กลุ่ม

### 7.1 กลุ่มระบบ

#### 🌐 ออนไลน์ (Google Sheets Sync)

เชื่อมต่อกับ Google Sheets ผ่าน Google Apps Script (GAS) เพื่อซิงค์ข้อมูลข้ามเครื่อง:
1. วาง **GAS Webhook URL** ในช่อง
2. กด **ทดสอบ** เพื่อยืนยันการเชื่อมต่อ
3. กด **สร้าง GAS Script** เพื่อรับโค้ดที่ต้องนำไป deploy ใน Google Apps Script
4. กด **Push ขึ้น** / **Pull ลง** เพื่อ sync ข้อมูลด้วยตนเอง
5. เปิด **Auto Sync** เพื่อให้ sync อัตโนมัติทุกครั้งที่มีการเปลี่ยนแปลง

#### 📱 LINE Notify

| ฟิลด์ | คำอธิบาย |
|------|---------|
| เปิดใช้งาน | toggle เปิด/ปิด LINE Notify |
| Channel Token | Token จาก LINE Notify (line.me/th/notify) |
| เวลาตรวจสอบ | ชั่วโมงที่แอปจะตรวจสอบและส่งแจ้งเตือนทุกวัน |
| กลุ่มเป้าหมาย | ดึงรายชื่อกลุ่ม LINE ที่ Token นี้เข้าถึงได้ |
| ทดสอบส่ง | ส่งข้อความทดสอบไปยังกลุ่มที่เลือก |

### 7.2 กลุ่มข้อมูลหลัก

#### 📦 กล่องยา

- เพิ่ม/ลบกล่อง
- แต่ละกล่องมี: รหัสกล่อง (ตัวเลขหรือข้อความ), ประเภท, สถานะเริ่มต้น
- ลบ = soft-delete (ไม่หายจากประวัติ)

#### 🗂 ประเภทกล่อง

- กำหนดชื่อประเภท, หมวดหมู่, อายุกล่อง (วัน)
- จัดการรายการยาใน template: ชื่อยา, จำนวนมาตรฐาน
- แต่ละยามี ID ถาวร — ไม่กระทบประวัติเมื่อแก้ชื่อ

#### 🏷 หมวดหมู่

จัดกลุ่มประเภทกล่อง เช่น CPR / สูติกรรม / ฉุกเฉิน / EMS

#### 🏥 ตึก / Ward

เพิ่ม/แก้ไขรายชื่อหอผู้ป่วยทั้งหมด (22 แห่ง seed ไว้แล้ว)

#### 👥 เจ้าหน้าที่

เพิ่ม/ลบชื่อเจ้าหน้าที่ แยกตามบทบาท:
- **เทคนิคเภสัชกรรม** — ผู้เตรียมยา
- **เภสัชกร** — ผู้ตรวจสอบ

### 7.3 กลุ่มค่าตั้ง

#### 🔔 แจ้งเตือน

| ค่า | ความหมาย |
|-----|---------|
| alertRed (วัน) | ยาหมดภายใน X วัน → สีแดง "วิกฤต" |
| alertYellow (วัน) | ยาหมดภายใน Y วัน → สีเหลือง "ระวัง" |
| boxExpireDays | อายุกล่อง default (วัน) ถ้าไม่กำหนดในประเภท |
| รูปแบบปีในแอป | พ.ศ. หรือ ค.ศ. สำหรับแสดงผลในหน้าจอ |
| รูปแบบปีในเอกสาร | พ.ศ. หรือ ค.ศ. สำหรับเอกสารพิมพ์ |
| ขนาดแสดงผล | Zoom 50%–100% (เฉพาะเครื่องนี้ ไม่ sync) |

#### 🖨 การพิมพ์

ตั้งค่าขนาดกระดาษ, font, ขอบกระดาษ สำหรับฉลากและใบปะหน้า

#### 🎨 เทมเพลตพิมพ์

ออกแบบ layout ฉลากกล่องแบบ drag-and-drop  
กำหนดตำแหน่งของ: รหัสกล่อง, ประเภท, วันหมดอายุ, รายการยา, QR Code, ลายเซ็น ฯลฯ

### 7.4 กลุ่มจัดการข้อมูล

#### 📥 นำเข้าข้อมูล

นำเข้าข้อมูลจากไฟล์ Excel (.xlsx):
- ข้อมูลกล่อง, ประเภท, ตึก, เจ้าหน้าที่
- ตรวจสอบรูปแบบก่อน import พร้อม preview

#### 💾 สำรองข้อมูล

- **Export** — ดาวน์โหลดข้อมูลทั้งหมดเป็นไฟล์ JSON
- **Import** — กู้คืนจากไฟล์ backup

#### 🔗 ฐานข้อมูล INVS

เชื่อมต่อระบบคลังยา INVS เพื่อดึงข้อมูล Lot/EXP ขณะบรรจุยา:
- ระบุ path ไฟล์ `invs.INI` (ไฟล์ config ของ INVS)
- กด **ทดสอบ** เพื่อยืนยันการเชื่อมต่อ
- เมื่อเชื่อมต่อสำเร็จ จะเห็น **Drug Mapping** — จับคู่ชื่อยาในกล่องกับรหัสยาใน INVS

## 8. การพิมพ์

BoxBox รองรับการพิมพ์ 2 ประเภท:

### ฉลากกล่อง (Label)

สติกเกอร์ขนาดเล็กติดที่กล่อง แสดง:
- รหัสกล่อง, ประเภท, วันบรรจุ, วันหมดอายุกล่อง
- QR Code สำหรับสแกนยืนยัน
- ชื่อผู้เตรียม / เภสัชกร

### ใบปะหน้ากล่อง (Cover Sheet)

ใบ A4 แสดงรายการยาทั้งหมดในกล่อง พร้อม:
- Lot No., EXP ของแต่ละรายการ
- จำนวนที่บรรจุ
- ลายเซ็นเภสัชกร

### QR Confirm Page

เมื่อพิมพ์ฉลาก ระบบสร้าง QR Code ที่ตึกสามารถสแกนเพื่อ:
- ดูรายละเอียดกล่องบนมือถือ
- กดยืนยันรับกล่อง (เปลี่ยนสถานะเป็น "ยืนยันแล้ว")

## 9. การเชื่อมต่อ INVS

BoxBox เชื่อมต่อกับระบบคลังยา INVS (HOSxP) โดยตรงผ่านไฟล์ `invs.INI`

### Smart Drug Matching

เมื่อค้นหายาจาก INVS ระบบใช้ AI จับคู่ชื่อยาอัตโนมัติ:

| วิธีจับคู่ | คำอธิบาย |
|-----------|---------|
| ★ Historical | เคยจับคู่ถูกต้องจาก dataset จริง — ความน่าเชื่อถือสูงสุด |
| 🟢 Score ≥ 80 | Token match สูง — แนะนำให้เลือก |
| 🟡 Score 60–79 | ตรวจสอบก่อนเลือก |
| ไม่มีผล | ชื่อยาไม่ตรงกับ INVS — ตั้งค่า Drug Mapping ด้วยตนเอง |

ระบบรองรับ:
- ชื่อ INN vs ชื่อทางการค้า (เช่น adrenaline → epinephrine)
- ตัวย่อเคมี (MgSO4, NaCl, KCl)
- การพิมพ์ผิดทั่วไป (tranesamic → tranexamic)
- ป้องกันการสลับยาที่คล้ายกัน (Ca-gluconate ≠ Ca-folinate)

### Drug Mapping (Settings → ฐานข้อมูล)

ตั้งค่าการจับคู่ชื่อยาด้วยตนเอง:
- **Auto Match** — ให้ระบบจับคู่ทั้งหมดอัตโนมัติ
- **ค้นหาเอง** — เลือก INVS item ให้ยาแต่ละตัว
- การ mapping sync ผ่าน Google Sheets ข้ามเครื่องได้
